What EAT is. EAT is a Center of Excellence body, not a project team. It sits above individual ERP/CRM testing engagements and owns the standards, tools, monitoring practices, capability programs and innovation pilots that every engagement draws on. Its output is a living repository: strategy documents, SOPs, templates, trackers and playbooks organized under five pillars, each broken into a discrete, ownable deliverable ("user story") with a named responsibility and clear acceptance criteria.

How it is built. The COE backlog is tracked exactly like a product backlog — 29 deliverables across 5 pillars, each carrying a status (Done / In Progress / To Do / Backlog), an owner role, and the artifacts that satisfy it. That backlog is the single source of truth behind the readiness dashboard and behind the folder structure of this site.

How it helps. A new ERP or CRM testing engagement should not have to write a test strategy, build a defect log template, or figure out a governance cadence from scratch. It opens the relevant pillar page here, pulls the current-version document or template, and adapts it — saving ramp-up time and keeping quality practices consistent across programs.

How this repository is organized

Five pillars → each pillar's numbered sub-areas (e.g. 1.1, 1.2 …) → each sub-area's page carries its purpose, owner, acceptance criteria and the actual documents/templates for that deliverable. The same five pillars, in the same order, are used across the tracker, the folder structure, and this site — so a reference like "1.4" means the same thing everywhere.
